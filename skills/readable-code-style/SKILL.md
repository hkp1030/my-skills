# Readable Code Style

짧고 바로 읽히는 코드를 우선하는 정리 원칙이다. 새 구조를 덧붙이기보다 이미 있는 흐름을 더 단순하고 선명하게 만드는 데 집중한다.

## 기본 원칙

- 과도한 구조화를 하지 않는다.
- 불필요한 방어 코드를 추가하지 않는다.
- 한눈에 읽히는 흐름을 최우선으로 둔다.
- 추상화보다 직접적인 표현을 먼저 선택한다.
- 예외 케이스가 실제로 필요한지 확인한 뒤에만 분기한다.

## 적용 방식

1. 먼저 현재 코드의 핵심 흐름을 한두 문장으로 설명할 수 있어야 한다.
2. 설명이 길어지면 분기, 중첩, 보조 함수, 래퍼 객체가 과한지 먼저 본다.
3. 같은 의미를 더 적은 개념 수로 표현할 수 있으면 그쪽을 선택한다.
4. 읽는 사람이 위에서 아래로 자연스럽게 따라갈 수 있도록 순서를 정리한다.

## 피할 것

- 의미 없이 레이어를 늘리는 헬퍼, 클래스, 설정 객체
- 실제 요구사항이 없는데 추가하는 null 체크, 재시도, fallback
- 짧은 로직을 오히려 따라가기 어렵게 만드는 추상화
- 이름만 일반화되고 사용처는 하나뿐인 공용 유틸

## 우선할 것

- 목적이 바로 보이는 변수명과 함수명
- 짧은 범위에서 끝나는 로직
- 조건 수가 적고 읽는 순서가 자연스러운 분기
- 중간 개념을 늘리지 않는 직접적인 데이터 흐름
